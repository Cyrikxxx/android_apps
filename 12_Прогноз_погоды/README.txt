ПРИЛОЖЕНИЕ 12. Прогноз погоды
=====================================

Что делает:
- Поиск погоды по названию города.
- Кнопка "Моё место" — погода по текущему местоположению.
- Показывает текущую температуру и прогноз на 5 дней (мин/макс температура
  и сумма осадков за день).
- Данные берутся с бесплатного API Open-Meteo (без ключа):
    * геокодинг города:  https://geocoding-api.open-meteo.com/v1/search
    * прогноз:            https://api.open-meteo.com/v1/forecast
- Сеть работает в отдельном Thread, интерфейс обновляется через runOnUiThread.

ПРО "КАРТУ ОСАДКОВ":
Полноценная карта осадков (радар на карте) требует картографического SDK
(Google Maps / OSM) и тайлов осадков — это выходит за рамки "без библиотек".
Поэтому вместо карты выводится сумма осадков (мм) по дням в прогнозе.

ЧТО ПОМЕНЯТЬ РУКАМИ
--------------------------------------------------
- package com.example.weather в MainActivity.java и в манифесте
  привести к пакету вашего проекта.

--------------------------------------------------
PERMISSIONS (ВАЖНО)
--------------------------------------------------
- INTERNET — обязательно для сетевых запросов.
- ACCESS_COARSE_LOCATION / ACCESS_FINE_LOCATION — для кнопки "Моё место".
  Эти два разрешения "опасные", поэтому кроме объявления в манифесте их
  надо запрашивать в РАНТАЙМЕ (код делает это в useMyLocation() через
  requestPermissions, а ответ ловит onRequestPermissionsResult).

--------------------------------------------------
НЮАНСЫ
--------------------------------------------------
- Android ЗАПРЕЩАЕТ сеть в главном потоке (NetworkOnMainThreadException),
  поэтому все запросы идут в new Thread(...).
- Местоположение берётся через getLastKnownLocation — оно может быть null
  (например, сразу после установки). Тогда приложение просит ввести город.
- minSdk рекомендуется 23+ (runtime-разрешения появились в API 23).

ПРИЛОЖЕНИЕ 14. Поиск и бронирование столиков
=====================================

Что делает:
- Экран со списком ресторанов (статические данные).
- По нажатию на ресторан — экран бронирования: выбор дня (ближайшие 5 дней)
  и свободного времени (уже занятые слоты не показываются).
- Кнопка "Забронировать" сохраняет бронь.
- Экран "Мои бронирования": список броней, долгое нажатие — отменить.
- Данные сохраняются (SharedPreferences, org.json).

--------------------------------------------------
КУДА КОПИРОВАТЬ ФАЙЛЫ
--------------------------------------------------
1) MainActivity.java     -> app/src/main/java/com/example/<пакет>/MainActivity.java
2) BookActivity.java     -> app/src/main/java/com/example/<пакет>/BookActivity.java
3) BookingsActivity.java -> app/src/main/java/com/example/<пакет>/BookingsActivity.java
4) activity_main.xml     -> app/src/main/res/layout/activity_main.xml
5) activity_book.xml     -> app/src/main/res/layout/activity_book.xml
6) activity_bookings.xml -> app/src/main/res/layout/activity_bookings.xml
7) AndroidManifest.xml   -> app/src/main/AndroidManifest.xml

--------------------------------------------------
ЧТО ПОМЕНЯТЬ РУКАМИ
--------------------------------------------------
- package com.example.restaurant во ВСЕХ трёх .java файлах и в манифесте
  привести к пакету вашего проекта. Все классы — в одной папке пакета.

--------------------------------------------------
PERMISSIONS
--------------------------------------------------
Не требуются (данные статические/локальные).

--------------------------------------------------
НЮАНСЫ
--------------------------------------------------
- В манифесте зарегистрированы ТРИ activity.
- Занятость слота определяется по совпадению ресторан+день+время среди
  сохранённых броней — это и есть "свободные слоты".
- minSdk 19+ (JSONArray.remove).

ПРИЛОЖЕНИЕ 9. Планировщик задач с календарём
=====================================

Что делает:
- Стандартный CalendarView для выбора дня.
- Добавление задачи: текст + время (TimePickerDialog).
- Список показывает задачи ТОЛЬКО выбранного в календаре дня.
- Долгое нажатие по задаче — удалить.
- При добавлении задачи показывается уведомление-напоминание.
- Данные сохраняются (SharedPreferences, JSON через org.json).

--------------------------------------------------
КУДА КОПИРОВАТЬ ФАЙЛЫ
--------------------------------------------------
1) MainActivity.java  -> app/src/main/java/com/example/<пакет>/MainActivity.java
2) activity_main.xml  -> app/src/main/res/layout/activity_main.xml
3) AndroidManifest.xml -> app/src/main/AndroidManifest.xml

--------------------------------------------------
ЧТО ПОМЕНЯТЬ РУКАМИ
--------------------------------------------------
- package com.example.planner в MainActivity.java и в манифесте
  привести к пакету вашего проекта.

--------------------------------------------------
PERMISSIONS / УВЕДОМЛЕНИЯ (ВАЖНО)
--------------------------------------------------
- В манифест добавлено:
    <uses-permission android:name="android.permission.POST_NOTIFICATIONS" />
  Это разрешение нужно с Android 13 (API 33). Кроме объявления в манифесте,
  его надо запросить в рантайме — код это делает в askNotificationPermission().
- С Android 8.0 (API 26) уведомления работают только через канал
  (NotificationChannel) — он создаётся в createChannel().
- Используется androidx.core (NotificationCompat) — это Jetpack, входит
  в стандартные проекты Android Studio (транзитивно через appcompat),
  отдельно ставить ничего не нужно.

--------------------------------------------------
НЮАНСЫ
--------------------------------------------------
- Уведомление показывается СРАЗУ при добавлении задачи (демонстрация
  механизма уведомлений). Чтобы уведомление приходило именно в назначенное
  ВРЕМЯ, нужно запланировать его через AlarmManager + BroadcastReceiver
  (setExactAndAllowWhileIdle) — это следующий шаг, здесь не делается,
  чтобы не усложнять.
- minSdk рекомендуется 21+. Каналы и разрешение обрабатываются проверками
  Build.VERSION.SDK_INT, так что на старых версиях код не упадёт.

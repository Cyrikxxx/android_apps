ПРИЛОЖЕНИЕ 13. Менеджер подписок
=====================================

Что делает:
- Добавление подписки: название, стоимость, дата списания (DatePickerDialog),
  период (Месяц / Год — Spinner).
- Список всех подписок с датой ближайшего списания.
- Считает суммарные траты В МЕСЯЦ и В ГОД (годовые подписки делятся на 12
  для месячной суммы, месячные умножаются на 12 для годовой).
- При запуске проверяет подписки со списанием в ближайшие 3 дня и шлёт
  уведомление.
- Долгое нажатие — удалить.
- Данные сохраняются (SharedPreferences, org.json).

--------------------------------------------------
КУДА КОПИРОВАТЬ ФАЙЛЫ
--------------------------------------------------
1) MainActivity.java  -> app/src/main/java/com/example/<пакет>/MainActivity.java
2) activity_main.xml  -> app/src/main/res/layout/activity_main.xml
3) AndroidManifest.xml -> app/src/main/AndroidManifest.xml

--------------------------------------------------
ЧТО ПОМЕНЯТЬ РУКАМИ
--------------------------------------------------
- package com.example.subs в MainActivity.java и в манифесте
  привести к пакету вашего проекта.

--------------------------------------------------
PERMISSIONS / УВЕДОМЛЕНИЯ
--------------------------------------------------
- POST_NOTIFICATIONS — нужно с Android 13 (объявлено в манифесте и
  запрашивается в рантайме в askNotificationPermission()).
- С Android 8.0 уведомления идут через канал (createChannel()).
- NotificationCompat — из Jetpack (androidx.core), входит в стандартные
  проекты Android Studio.

--------------------------------------------------
НЮАНСЫ
--------------------------------------------------
- Уведомление "за 3 дня" проверяется при ЗАПУСКЕ приложения (проверка diff
  между датой списания и текущим временем). Чтобы уведомление приходило
  само в фоне, даже когда приложение закрыто, нужен AlarmManager +
  BroadcastReceiver (или WorkManager) — здесь для простоты не делается.
- minSdk рекомендуется 21+.
